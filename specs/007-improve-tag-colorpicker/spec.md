# Feature Specification: プレイヤータグ編集画面のカラーピッカー改善

**Feature Branch**: `007-improve-tag-colorpicker`
**Created**: 2026-04-10
**Status**: Draft
**Input**: User description: "プレイヤータグ編集画面におけるカラーピッカーを改善する。文字を用いず色だけを羅列して選択するカラーピッカー専用コンポーネントを作成し、タグ編集画面で利用する。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 色スウォッチでのカラー選択 (Priority: P1)

ユーザーはプレイヤータグを作成・編集する際に、色名のテキストではなく色そのものを視覚的に確認しながら直感的に色を選択できる。カラーピッカーは色のスウォッチ（丸や正方形などの色付きブロック）を並べて表示し、ユーザーはクリックまたはタップで色を選択できる。

**Why this priority**: カラーピッカーの改善はタグの作成・編集フローに直接影響し、全ユーザーが使う機能であるため最優先。現状の実装では選択中の色が視覚的に不明確で、色名テキストがボタン内に表示されて見づらいという問題がある。

**Independent Test**: タグ作成ダイアログを開き、専用カラーピッカーコンポーネントが色スウォッチを並べて表示することを確認し、各スウォッチをクリックすると選択状態が明確に変わることを確認する。

**Acceptance Scenarios**:

1. **Given** タグ作成・編集ダイアログが開いている, **When** カラーピッカーを表示する, **Then** 利用可能な8色のスウォッチが色名テキストなしで視覚的に並んでいる
2. **Given** カラーピッカーが表示されている, **When** スウォッチをクリック・タップする, **Then** 選択されたスウォッチが視覚的に選択状態（枠・チェックマーク・拡大など）で強調表示される
3. **Given** カラーピッカーで色を選択した, **When** タグを保存する, **Then** 選択した色がタグに適用されて表示される
4. **Given** 既存タグを編集するダイアログを開く, **When** カラーピッカーが表示される, **Then** 現在のタグの色に対応するスウォッチが選択状態で表示される
5. **Given** カラーピッカーのスウォッチにフォーカスを当てる, **When** キーボードで操作する, **Then** 矢印キー等で選択を移動でき、エンターキーで色を確定できる（アクセシビリティ確保）

---

### Edge Cases

- カラーピッカーに渡された現在の色値がプリセットカラーリストに存在しない場合でも、UIがクラッシュせずデフォルト色（gray）の選択状態を表示する
- スウォッチをキーボードのみで操作できる（スクリーンリーダー対応を含む）
- 各スウォッチに適切なaria-labelが付与されており、色の名前がスクリーンリーダーで読み上げられる
- ダイアログを閉じて再度開いた場合に、前回の選択状態が適切にリセット（または維持）される

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムはプレイヤータグの色を選択するための専用カラーピッカーコンポーネントを提供しなければならない
- **FR-002**: カラーピッカーは、色名テキストを一切表示せず、色のスウォッチ（色付きブロック）のみを並べて表示しなければならない
- **FR-003**: カラーピッカーは利用可能な8種類のプリセットカラー（gray、red、orange、yellow、green、blue、purple、pink）をすべて表示しなければならない
- **FR-004**: 選択中のスウォッチは未選択のスウォッチと視覚的に明確に区別できる状態表示を持たなければならない
- **FR-005**: カラーピッカーは現在選択中の色を外部から受け取り、対応するスウォッチを選択状態で初期表示しなければならない
- **FR-006**: ユーザーがスウォッチを選択すると、選択色の変更が親コンポーネントに通知されなければならない
- **FR-007**: カラーピッカーはキーボード操作をサポートし、すべてのスウォッチにアクセス可能でなければならない
- **FR-008**: 各スウォッチにはスクリーンリーダーが読み上げ可能な色名ラベル（aria-label）を付与しなければならない
- **FR-009**: プレイヤータグ作成・編集ダイアログは既存の汎用トグルグループコンポーネントの代わりに専用カラーピッカーコンポーネントを使用しなければならない

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: カラーピッカーにおいて、選択中の色と未選択の色が1秒以内に視覚的に識別できる（ユーザーテストで90%以上が正しく選択状態を認識できる）
- **SC-002**: タグ編集ダイアログを開いた際に、現在の色が即座にカラーピッカーで選択状態として表示される
- **SC-003**: キーボードのみでカラーピッカーのすべての色を選択操作できる
- **SC-004**: カラーピッカーコンポーネントが再利用可能な独立したコンポーネントとして実装され、将来の画面でも利用できる

## Assumptions

- 利用可能なカラーセットは既存の8色（gray、red、orange、yellow、green、blue、purple、pink）から変更しない
- カスタムカラー（カラーホイール等）は本機能のスコープ外とする
- 色のラベルは既存のカラー定数に含まれる英語名（color name）をそのままaria-labelとして使用する
- スウォッチの形状（丸・正方形など）は実装時に決定してよい
