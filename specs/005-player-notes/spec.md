# Feature Specification: プレイヤーメモ機能

**Feature Branch**: `005-player-notes`
**Created**: 2026-03-28
**Status**: Draft
**Input**: User description: "プレイヤーメモ機能を実装したいです。プレイヤーに任意数のタグを割り当て、エディタを使ってHTML形式でメモを作成できるようにしたいです"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - プレイヤーの登録と管理 (Priority: P1)

ユーザーは対戦相手や同卓プレイヤーの情報を記録するために、プレイヤーを新規登録し、一覧で管理できる。プレイヤー名を入力して登録し、登録済みプレイヤーを一覧から検索・閲覧できる。

**Why this priority**: プレイヤーエンティティはタグやメモの基盤となるため、最も優先度が高い。プレイヤーが存在しなければ他の機能は成り立たない。

**Independent Test**: プレイヤーを新規登録し、一覧画面で確認できれば、この機能単体で「対戦相手リスト」として価値を提供できる。

**Acceptance Scenarios**:

1. **Given** ユーザーがログイン済みである, **When** プレイヤー名を入力して登録する, **Then** プレイヤーが一覧に表示される
2. **Given** 複数のプレイヤーが登録されている, **When** 一覧画面を開く, **Then** 登録済みのすべてのプレイヤーが表示される
3. **Given** プレイヤーが登録されている, **When** プレイヤー名を編集する, **Then** 更新された名前が一覧に反映される
4. **Given** プレイヤーが登録されている, **When** プレイヤーを削除する, **Then** 一覧からそのプレイヤーが消え、関連するタグ・メモも削除される

---

### User Story 2 - プレイヤーへのタグ割り当て (Priority: P2)

ユーザーはプレイヤーの特徴やプレイスタイルを分類するために、任意の数のタグをプレイヤーに割り当てることができる。タグは自由に作成でき、色を設定して視覚的に区別できる。複数プレイヤーに同じタグを使い回せる。

**Why this priority**: タグはプレイヤーの素早い分類・検索を可能にし、メモより軽量な情報記録手段として高頻度で使われる。

**Independent Test**: プレイヤーを登録し、「タイト」「アグレッシブ」などのタグを付与して、タグで絞り込み検索ができれば単体で価値がある。

**Acceptance Scenarios**:

1. **Given** プレイヤーが登録されている, **When** 新しいタグ名を入力して割り当てる, **Then** そのタグがプレイヤーに表示される
2. **Given** プレイヤーに複数のタグが割り当てられている, **When** プレイヤー詳細を表示する, **Then** すべてのタグが色付きで表示される
3. **Given** 既存のタグが存在する, **When** 別のプレイヤーにそのタグを割り当てる, **Then** 同じタグが複数のプレイヤーで共有される
4. **Given** プレイヤーにタグが割り当てられている, **When** タグを削除する, **Then** そのプレイヤーからタグが外れるが、タグ自体は他のプレイヤーで引き続き使用可能である
5. **Given** 複数のプレイヤーが登録されている, **When** 特定のタグでフィルタリングする, **Then** そのタグが割り当てられたプレイヤーのみが表示される
6. **Given** タグが作成されている, **When** タグの色を変更する, **Then** そのタグが使われているすべての箇所で新しい色が反映される

---

### User Story 3 - リッチテキストエディタによるメモ作成 (Priority: P3)

ユーザーはプレイヤーに関する詳細な観察やメモを、リッチテキストエディタを使ってHTML形式で作成・編集できる。太字、見出し、リスト、リンクなどの書式を使い、読みやすいメモを残せる。

**Why this priority**: メモはプレイヤーに関する詳細な情報を構造化して記録する機能で、タグよりも詳しい情報を残す際に使用される。

**Independent Test**: プレイヤーを登録し、リッチテキストエディタでメモを作成・保存して、後から閲覧・編集できれば単体で価値がある。

**Acceptance Scenarios**:

1. **Given** プレイヤーが登録されている, **When** メモ作成エディタを開く, **Then** リッチテキストエディタが表示され、書式付きテキストを入力できる
2. **Given** エディタが開いている, **When** 太字・見出し・リスト・リンクなどの書式を適用する, **Then** プレビューで書式が正しく反映される
3. **Given** メモを入力した, **When** 保存する, **Then** HTML形式でメモが保存され、次回閲覧時に書式が保持されている
4. **Given** 保存済みのメモがある, **When** メモを編集する, **Then** 既存の書式が維持された状態でエディタが開き、追記・修正できる
5. **Given** プレイヤーにメモが保存されている, **When** プレイヤー詳細を表示する, **Then** 書式付きのメモが読みやすく表示される

---

### Edge Cases

- プレイヤー名が空または空白のみの場合、登録を拒否する
- 同一ユーザーが同じプレイヤー名を重複登録しようとした場合の挙動（許可する：同姓同名の別人を想定）
- 非常に長いメモ（HTMLコンテンツ）を保存する場合の上限
- タグ名が空または空白のみの場合、割り当てを拒否する
- プレイヤー削除時に関連するタグ関連付けとメモが適切にクリーンアップされる
- リッチテキストエディタでの不正なHTML（スクリプトタグなど）のサニタイズ
- メモが未作成のプレイヤーでも正常に表示される

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは、ユーザーが名前を指定してプレイヤーを新規登録できなければならない
- **FR-002**: システムは、登録済みプレイヤーの一覧表示・検索機能を提供しなければならない
- **FR-003**: システムは、プレイヤー名の編集と、プレイヤーの削除をサポートしなければならない
- **FR-004**: システムは、ユーザーがタグを名前と色を指定して作成し、任意のプレイヤーに割り当てられなければならない
- **FR-004a**: タグにはプリセットカラーパレットから色を選択できなければならない
- **FR-004b**: タグの色は作成後に変更可能でなければならない
- **FR-005**: システムは、1人のプレイヤーに複数のタグを割り当て可能でなければならない
- **FR-006**: システムは、同一タグを複数のプレイヤーに割り当て可能でなければならない（多対多の関係）
- **FR-007**: システムは、タグによるプレイヤーのフィルタリング機能を提供しなければならない
- **FR-008**: システムは、リッチテキストエディタを通じてHTML形式のメモを作成・編集できなければならない
- **FR-009**: リッチテキストエディタは、少なくとも太字、斜体、見出し、箇条書きリスト、番号付きリスト、リンクの書式をサポートしなければならない
- **FR-010**: システムは、保存されたHTMLメモを書式付きで正しくレンダリングしなければならない
- **FR-011**: システムは、HTMLメモに含まれる不正なスクリプトやイベントハンドラをサニタイズしなければならない
- **FR-012**: プレイヤーとその関連データ（タグの関連付け、メモ）はユーザーごとに分離されなければならない
- **FR-013**: プレイヤーを削除した場合、関連するタグの関連付けとメモもカスケード削除されなければならない

### Key Entities

- **プレイヤー (Player)**: 対戦相手や同卓のプレイヤーを表す。名前を持ち、ユーザーに所属する。タグやメモと関連付けられる。
- **プレイヤータグ (PlayerTag)**: プレイヤーの特徴やプレイスタイルを分類するためのラベル。名前と色を持つ。ユーザーに所属し、複数のプレイヤーと多対多の関係を持つ。
- **プレイヤーメモ (PlayerMemo)**: プレイヤーに関する詳細な観察や情報をHTML形式で保存するメモ。1プレイヤーにつき1つのメモを持つ（上書き更新方式）。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ユーザーがプレイヤーを30秒以内に新規登録できる
- **SC-002**: 50人以上のプレイヤーが登録されていても、一覧の表示と検索が1秒以内に完了する
- **SC-003**: タグの割り当て・解除操作が2タップ以内で完了できる
- **SC-004**: リッチテキストエディタでのメモ作成・保存が直感的に行え、書式の適用に迷わない
- **SC-005**: 保存したメモを再度開いた際に、すべての書式が正しく保持されている
- **SC-006**: プレイヤーデータが他のユーザーから完全に分離されている

## Assumptions

- プレイヤー名の重複は許可する（同姓同名の別人を想定）
- プレイヤーは認証済みユーザーのみが管理できる（既存の認証基盤を使用）
- 1プレイヤーにつきメモは1つ（複数メモではなく単一メモの上書き更新方式）
- タグの色はプリセットカラーパレットから選択する方式とする（カスタムカラーピッカーは初期スコープに含めない）
- タグのアイコン設定は初期スコープに含めない
- プレイヤーとポーカーセッションの紐付けは本機能のスコープ外とする
- メモの文字数上限はシステムの標準的な制限に従う
- リッチテキストエディタはWYSIWYG形式とする
